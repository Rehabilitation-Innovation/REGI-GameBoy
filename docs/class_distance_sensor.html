◆ _sensor_map

DistanceSensor * DistanceSensor::_sensor_map = {}
staticprotected

My solution to the "C++ IRQ problem" The problem is that an IRQ needs to be a plain C function, so I can't have a function that modifies the running instance of the DistanceSensor that triggered the request. There's no way for the IRQ to know which sensor sent the request.

But I can store the instance of the distance sensor in a map, so the sensor running on PIO 0, state machine 1 will map to _sensor_map[0][1]. Thus the ISR can access the sensor instance to provide a reading

Instance of map that stores the memory address of the loaded pio program
